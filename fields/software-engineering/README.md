## Overview

The Pragmatic Engineer AI Tooling Survey (March 2026, 900+ respondents) confirmed what practitioners already felt: **AI is now central to software engineering**, not supplementary. 95% use AI tools weekly, 56% do 70%+ of their work with AI, and 55% regularly use autonomous AI agents. The shift from "copilot" (AI suggests completions) to "agent" (AI autonomously executes multi-step tasks) is THE story of 2025-2026.

Staff+ engineers are the heaviest agent users (63.5%), suggesting that experience amplifies AI leverage. Claude Code went from zero to #1 tool in 8 months. Most engineers juggle 2-4 tools simultaneously.

## ✅ What AI Can Do

<table>
<tr>
<td width="50%" valign="top">

#### Fully Automated
*AI does the work, humans review.*

- 💻 **Code completion & boilerplate** — Standard patterns, scaffolding, CRUD
- 🧪 **Test generation** — Unit tests, edge cases, test scaffolding
- 📝 **Documentation** — Docstrings, API docs, README files
- 🐛 **Simple bug fixes** — Off-by-one, null checks, typos
- 🔄 **Code translation** — Between languages with high accuracy
- 💬 **Commit messages & PR descriptions** — Accurate change summaries

</td>
<td width="50%" valign="top">

#### Partially Automated
*Humans direct, AI produces drafts.*

| Task | AI Handles | Human Handles |
|:-----|:-----------|:-------------|
| Code review | Common issues, style, patterns | Architectural fit, team context |
| Debugging | Suggests fixes, explains errors | Validates root cause, system context |
| Refactoring | Mechanical transformations | When/why to refactor |
| Feature impl. | Code generation from specs | Requirements, validation, intent |
| PR descriptions | Auto-generated summaries | Context, motivation, risk notes |

</td>
</tr>
</table>

## ❌ What Still Requires Humans

| Task | Why AI Can't Do This |
|:-----|:---------------------|
| **System architecture** | Business context + scalability + team capabilities + operational constraints simultaneously |
| **Business requirements** | Translating vague stakeholder needs into specs requires empathy and domain knowledge |
| **Organizational trade-offs** | Decisions shaped by team politics, budget, timelines, strategy |
| **Security-critical judgment** | Threat modeling and risk assessment require adversarial thinking in context |
| **Novel algorithm design** | Creating algorithms for unprecedented problems requires creative leaps |
| **Production incidents** | Debugging distributed systems requires deployment topology and cross-team context |
| **Team convention review** | Understanding why code exists — past incidents, team agreements, legacy constraints |
